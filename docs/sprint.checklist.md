# Checklist de refonte UI/UX (ordre recommandé)

## Sprint 1 — Fondation visuelle & conversion (Home)

- [ ] Clarifier la promesse principale du site (1 phrase orientée résultat client).
- [ ] Revoir la hiérarchie de la Home : Hero → preuves → offres → méthode → FAQ → CTA.
- [ ] Uniformiser les composants clés (boutons, cartes, badges, sections).
- [ ] Renforcer la crédibilité visible (chiffres, projets, résultats, témoignages).
- [ ] Valider la lisibilité mobile (espaces, tailles de texte, ordre des sections).
- [ ] QA accessibilité de base (focus visible, contrastes, structure Hn).

## Sprint 2 — Système design global

- [ ] Finaliser les tokens UI (couleurs, rayons, ombres, spacing, typo).
- [ ] Créer des variants réutilisables (Primary CTA, Secondary CTA, card default, card feature).
- [ ] Harmoniser Header + Footer + transitions entre sections.
- [ ] Définir un style iconographique cohérent.
- [ ] Nettoyer les incohérences visuelles inter-pages.

## Sprint 3 — Pages Offres (conversion)

- [ ] Clarifier chaque offre : pour qui, livrables, délais, tarif, CTA.
- [ ] Ajouter des preuves proches de l’offre (cas client ou résultat associé).
- [ ] Réduire la friction décisionnelle (FAQ ciblée, comparatif simple).
- [ ] Optimiser les CTA de prise de contact / devis.

## Sprint 4 — Pages Projets (autorité)

- [ ] Structurer chaque projet en storytelling : contexte → problème → solution → impact.
- [ ] Standardiser les composants d’étude de cas.
- [ ] Renforcer la preuve chiffrée et les avant/après.
- [ ] Ajouter des liens croisés vers offres pertinentes.

## Sprint 5 — Méthode & À propos (confiance)

- [ ] Rendre la méthode plus concrète avec livrables par étape.
- [ ] Transformer la page À propos en preuve de posture + expertise.
- [ ] Ajouter les éléments de réassurance (process, cadre, collaboration).

## Sprint 6 — Contact / Devis (taux de prise de RDV)

- [ ] Simplifier les formulaires (moins de champs, meilleure guidance).
- [ ] Clarifier ce qui se passe après soumission.
- [ ] Mettre en avant les délais de réponse et modalités de collaboration.
- [ ] Tester les CTA sur mobile (zone cliquable, ordre visuel).

## Sprint 7 — Qualité finale & lancement

- [ ] Passe responsive complète (mobile, tablette, desktop large).
- [ ] Optimisation performance (images, poids JS, sections critiques).
- [ ] Passe SEO on-page (titres, métas, structure des contenus).
- [ ] Contrôle accessibilité (navigation clavier, labels, contrastes).
- [ ] Validation analytics (événements CTA/formulaires).
- [ ] Check final avant publication (contenu, liens, cohérence visuelle).
